Flagship Engagement

Etihad Rail — Multi-Site Rail & Terminal Operations Platform

National rail operator · 5 live terminal sites · AWS EKS · Actively maintained through 2025

Took over full technical ownership of Etihad Rail's multi-terminal logistics platform — 40+ microservice repositories running across 5 live sites — following a vendor transition, and have operated, extended, and modernized it since. The platform coordinates rail and truck gate operations, vehicle booking, yard tracking, and rail gateways into one centrally-managed system with a live control-tower dashboard.

Highlights
  • Assumed full technical ownership of a 40+ microservice platform after vendor handover, including the complete Kubernetes deployment lifecycle across 5 physical sites on AWS EKS
  • Designed and built 6 independent .NET Core integration services (Clean/Onion architecture, CQRS with MediatR, message-driven via MassTransit/RabbitMQ)
  • Automated multi-site deployment with a multi-repo Helm chart pipeline for repeatable rollouts across mixed Windows/Linux workloads
  • Built a custom Vue 3 + TypeScript control-tower dashboard with Arabic/RTL support and geospatial yard visualization
  • Implemented platform-wide SSO (Keycloak) as version-controlled infrastructure, securing staff access and service-to-service integration
  • Delivered centralized observability (ELK + Prometheus/Grafana) with live per-site dashboards for yard occupancy, turnaround time, and container throughput
  • Backed by a dedicated system-level test suite validating cross-service integration end-to-end
Systems Integrated
  • Central Terminal Operating System (CTOS) — the control-tower hub aggregating operations from all 5 sites into one live view
  • Local Terminal Operating System (LTOS) — per-site gateway layer connecting local operations back to the central hub
  • Train Management System — real-time sync between the national train management system and terminal gate operations
  • Vehicle Booking System — automated slot booking and truck arrival coordination at each site's gate
  • Gate Operating System — truck and rail gate clearance and access control at every terminal
  • DGPS tracking — real-time differential-GPS positioning feeding yard equipment and vehicle tracking

More from the Smart Port Management platform

Berth Planning
Problem

Terminal operators were scheduling vessel berths manually, causing double-bookings and no real-time visibility into bollard availability.

Solution

A time-based berth scheduling system with vessel assignments, bollard allocation, and live arrival/departure tracking.

Stack

.NET Core · React · PostgreSQL · RabbitMQ · Kubernetes

3D Berth Monitoring
Problem

Terminal staff had no live visibility into vessel positioning and bollard occupancy across the berth in real time.

Solution

A 3D real-time berth occupancy view with live vessel positioning and bollard assignments, streamed over WebSockets.

Stack

.NET Core · Vue.js · WebSockets · Kubernetes

GIS Zone Management
Problem

Terminal layout and zone definitions were static and hard to reconfigure as operations and map data changed.

Solution

An interactive GIS-based zone definition and terminal layout configuration tool built directly on live port maps.

Stack

GIS/Mapping · .NET Core · Vue.js · PostgreSQL
